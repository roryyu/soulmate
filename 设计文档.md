## 2. 数据库设计 (Prisma Schema)

采用模块化命名设计。`User` 是全局的，而 `ResearchProject` 专门属于"Soulmates"模块。

```prisma
generator client {
  provider = "prisma-client-js"
}

datasource db {
  provider = "postgresql"
  url      = env("DATABASE_URL")
}

// ========== 全局平台层 ==========

model User {
  id               String            @id @default(cuid())
  email            String            @unique
  password         String
  name             String?
  role             String            @default("TEACHER") // TEACHER, ADMIN
  createdAt        DateTime          @default(now())
  
  // 关联：立项宝模块的数据
  researchProjects ResearchProject[]
  
  // 未来关联：
  // lessonPlans   LessonPlan[]
}

// ========== 模块 1: 立项宝 (Research Pro) ==========

// 课题项目表
model ResearchProject {
  id          String   @id @default(cuid())
  userId      String
  title       String   // 课题名称
  field       String   // 学科领域 (如：小学数学)
  description String?  // 初始构思
  status      String   @default("DRAFT") // DRAFT, COMPLETED
  createdAt   DateTime @default(now())
  updatedAt   DateTime @updatedAt
  
  user        User     @relation(fields: [userId], references: [id], onDelete: Cascade)
  
  // 子功能数据
  ideas       ResearchIdea[]
  searches    ResearchSearch[]
  references  ResearchReference[]
}

model ResearchIdea {
  id          String   @id @default(cuid())
  projectId   String
  title       String
  rationale   String   @db.Text
  isAdopted   Boolean  @default(false)
  createdAt   DateTime @default(now())
  
  project     ResearchProject @relation(fields: [projectId], references: [id], onDelete: Cascade)
}

model ResearchSearch {
  id            String   @id @default(cuid())
  projectId     String
  userTopic     String
  cnkiQuery     String
  createdAt     DateTime @default(now())

  project       ResearchProject @relation(fields: [projectId], references: [id], onDelete: Cascade)
}

model ResearchReference {
  id               String   @id @default(cuid())
  projectId        String
  fileName         String
  summary          String   @db.Text
  innovationPoints String   @db.Text
  methodology      String?  @db.Text
  keyPages         String?
  createdAt        DateTime @default(now())

  project          ResearchProject @relation(fields: [projectId], references: [id], onDelete: Cascade)
}

```

## 4. 音乐处理库 (Music Processing Libraries)

项目包含三个核心音乐处理库，用于音乐生成、预处理和编辑。

### 4.1 音乐生成库 (`lib/music.ts`)

**功能**: 封装 Minimax 音乐生成 API，支持通过提示词、歌词、参考音频等方式生成音乐。

**安装依赖**:
```bash
npm install fluent-ffmpeg ffmpeg-static ffprobe-static uuid
npm install --save-dev @types/uuid
```

**使用示例**:
```typescript
import { generateMusic, MusicGenerationRequest } from '@/lib/music';

// 基础使用：通过提示词生成音乐
const result = await generateMusic({
  prompt: "轻快的钢琴曲，适合儿童教育",
  output_format: "url",
  is_instrumental: true,
});

// 高级使用：带歌词和音频设置
const result = await generateMusic({
  prompt: "温馨的儿歌",
  lyrics: "太阳当空照，花儿对我笑...",
  audio_setting: {
    sample_rate: 44100,
    bitrate: 320,
    format: "mp3"
  }
});

console.log("生成的音乐URL:", result.data.audio);
console.log("音乐时长:", result.extra_info.music_duration, "ms");
```

### 4.2 音乐母带预处理库 (`lib/music-pre.ts`)

**功能**: 对上传的音乐进行预处理，提取歌词、结构分析、封面特征等。

**使用示例**:
```typescript
import { preprocessMusicCover } from '@/lib/music-pre';

const result = await preprocessMusicCover({
  audio_url: "https://example.com/song.mp3"
});

console.log("封面特征ID:", result.cover_feature_id);
console.log("格式化歌词:", result.formatted_lyrics);
console.log("结构分析结果:", result.structure_result);
console.log("音频时长:", result.audio_duration, "秒");
```

### 4.3 音乐控制编辑库 (`lib/music-control.ts`)

**功能**: 支持多首音乐的截取、拼接、按时间轴插入，自动处理音轨间的沉默间隙。

**核心功能**:
- ✅ 音乐文件截取（指定开始/结束时间）
- ✅ 多音轨按时间轴插入
- ✅ 自动填充沉默间隙
- ✅ 最终音频拼接
- ✅ MP3 格式输出
- ✅ 完全使用 Base64 格式（输入输出）

**使用示例**:
```typescript
import { controlMusic, mergeMusicFiles } from '@/lib/music-control';

// 方式一：使用完整接口
const mergedBase64 = await controlMusic({
  musicFiles: [
    {
      fileData: "base64_string_of_intro",  // base64格式的音频
      startTime: 0,      // 从0秒开始
      endTime: 15,      // 截取到15秒
      insertTime: 0,    // 在0秒处插入
    },
    {
      fileData: "base64_string_of_main",
      startTime: 5,
      endTime: 35,
      insertTime: 20,   // 在20秒处插入（中间有5秒沉默过渡）
    },
    {
      fileData: "base64_string_of_outro",
      insertTime: 60,   // 在60秒处插入
    }
  ],
  duration: 90,       // 总时长90秒（超出部分自动填充沉默）
});

// 方式二：使用简化接口
const mergedBase64 = await mergeMusicFiles([
  {
    data: "base64_string_of_song1",
    trimStart: 0,
    trimEnd: 30,
    insertAt: 0,
  },
  {
    data: "base64_string_of_song2",
    trimStart: 10,
    trimEnd: 40,
    insertAt: 35,
  }
], 80);

console.log("合并后的Base64:", mergedBase64);
```

**说明**:
- 临时文件默认存储在 `temp/music-control/` 目录
- 支持 MP3、WAV、M4A、FLAC 等常用音频格式
- 文件大小建议不超过 50MB
- 自动处理文件不存在、时间越界等异常情况
- 输入输出均为 Base64 字符串格式

## 5. 音乐项目功能更新 (Music Project Features)

**功能**: 在创建音乐项目时，自动调用 Minimax 音乐生成 API。

- 调用 `generateMusic` 函数生成新音乐
- 输出详细的调用和返回日志

**调用参数**:
```typescript
{
  prompt: "...",              // 提示词
  output_format: "url",       // 输出格式

  is_instrumental: true,      // 器乐

    sample_rate: 44100,
    bitrate: 256000,
```

## 7. 开发实施步骤 (Step-by-Step Implementation)

### 第一步：基础架构与平台层
1.  **项目初始化**: Next.js App Router, Tailwind, Shadcn UI。
2.  **数据库**: 运行 `npx prisma db push`。
3.  **路由规划 (App Router 结构)**:
    *   `app/page.tsx`: **平台首页 (Launcher)**。
        *   展示 Grid 布局的应用入口：
            *   [图标] 立项宝 (点击跳转 `/admin/prescription`)
            *   [图标] 智能批改 (禁用状态，显示 Coming Soon)
            *   [图标] AI 备课 (禁用状态，显示 Coming Soon)
    *   `app/auth/...`: 处理登录注册。

### 第二步：立项宝 (Research Pro) 后端逻辑
创建 `actions/research/` 文件夹（命名空间隔离）：
1.  `project-actions.ts`:
    *   `createResearchProject()`
    *   `getResearchProjects()` (注意：Filter by userId)
2.  `ai-actions.ts`:
    *   **Context 注入**: 获取 `ResearchProject` 的 `title` + `field` 作为背景信息。
    *   `generateIdeas`: 调用 `RESEARCH_PROMPTS.IDEATION`。

### 第三步：立项宝 (Research Pro) 前端实现
所有路由都在 `app/research/` 下：

1.  **Dashboard**: `app/admin/prescription/page.tsx`
    *   面包屑导航：平台首页 > 立项宝。
    *   功能：查看历史课题列表，新建处方。

2.  **工作流布局**: `app/research/[projectId]/layout.tsx`
    *   **侧边栏菜单**:
        1.  🎯 选题构思 (Ideation)
        2.  🔍 检索助手 (Search)
        3.  📖 文献速读 (Reading)

3.  **核心页面**:
    *   **选题构思 (`/ideation`)**: 交互式 Chat 界面。用户输入："我想研究小学数学的错题管理"。AI 返回 5 个精准题目。
    *   **检索助手 (`/search`)**: 提供"一键复制"功能，方便老师直接去知网粘贴。

### 第四步：UI/UX 细节
*   **平台感**: 顶栏 (Navbar) 显示 "智教 AI (EduNexus)" Logo 和 用户头像。
*   **返回机制**: 在“立项宝”界面左上角增加 "返回应用中心" 按钮。
*   **状态反馈**: AI 生成通常需要 10-30 秒，使用 `Skeleton` 骨架屏或 Loading Spinner 缓解焦虑。
*   **移动端适配**: 考虑到老师可能在手机查看，侧边栏在移动端应折叠为汉堡菜单。
